# PARTE I — APP DE NOTAS (BASE PROFESIONAL)

## Objetivo general

Construir una aplicación web de notas tipo chat con una arquitectura sólida, escalable y preparada para futuras expansiones como:

* sincronización entre dispositivos
* aplicación mobile
* manejo de archivos
* múltiples usuarios
* múltiples temas visuales
* internacionalización

Sin implementar todavía:

* sincronización real
* versión mobile
* adjuntos funcionales
* WebSockets
* Kafka
* Kubernetes

La meta principal es crear una base limpia y profesional.

---

# Características importantes de la DB

## UUID

Todas las tablas principales usarán UUID.

Beneficios:

* evita colisiones
* compatible con sync futura
* compatible multi dispositivo

---

## Timestamps

Campos:

* creado_en
* actualizado_en
* eliminado_en
* ultima_modificacion

Beneficios:

* auditoría
* sincronización futura
* consistencia

---

## Soft Delete

Uso de:

```txt
eliminado_en
```

Beneficios:

* recuperación de datos
* sync futura
* seguridad

---

## Versionado

Uso de:

```txt
version
```

Beneficios:

* control de concurrencia
* sincronización futura
* consistencia entre dispositivos

# Modelo Mermaid

```mermaid
erDiagram

    USUARIO ||--o{ NOTA : posee
    USUARIO ||--o{ CATEGORIA : crea
    USUARIO ||--o{ USUARIO_ROL : tiene

    ROL ||--o{ USUARIO_ROL : asignado
    ROL ||--o{ ROL_PERMISO : tiene

    PERMISO ||--o{ ROL_PERMISO : asignado

    CATEGORIA ||--o{ NOTA : agrupa
    IDIOMA ||--o{ NOTA : define
    NOTA ||--o{ ADJUNTO : contiene


    USUARIO {
        uuid id
        string email
        string password

        datetime creado_en
        datetime actualizado_en
        datetime eliminado_en
    }


    ROL {
        uuid id
        string nombre
    }


    PERMISO {
        uuid id
        string nombre
    }


    USUARIO_ROL {
        uuid usuario_id
        uuid rol_id
    }


    ROL_PERMISO {
        uuid rol_id
        uuid permiso_id
    }


    CATEGORIA {
        uuid id

        string nombre
        string color

        uuid usuario_id

        datetime creado_en
        datetime actualizado_en
    }


    IDIOMA {
        uuid id

        string codigo
        string nombre
    }


    NOTA {
        uuid id

        uuid usuario_id
        uuid categoria_id
        uuid idioma_id

        text contenido

        int version

        datetime creado_en
        datetime actualizado_en
        datetime eliminado_en
        datetime ultima_modificacion
    }


    ADJUNTO {
        uuid id

        uuid nota_id

        string nombre_archivo
        string tipo_archivo
        string url_archivo

        int orden

        datetime creado_en
        datetime eliminado_en
    }
```
